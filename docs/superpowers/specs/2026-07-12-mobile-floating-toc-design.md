# 移动端圆形悬浮目录设计

## 目标

让手机端长文章的目录在任意阅读位置都可访问，同时把常驻控件对正文的遮挡降到最低。桌面端现有左侧 sticky 目录保持不变。

## 适用范围

- 仅在文章存在至少 3 个 H2/H3 标题且视口宽度不超过 900px 时启用。
- 移除移动端正文顶部的静态目录卡片。
- 不修改文章内容、标题层级、多语言路由或桌面端目录。

## 收起状态

- 在右下角显示一个 `48px × 48px` 的圆形目录按钮。
- 距右侧 `16px`，距底部为 `16px + env(safe-area-inset-bottom)`。
- 按钮只显示项目现有图标体系中的目录图标，不显示章节标题或进度。
- 按钮使用轻量毛玻璃材质：半透明中性色背景、约 `20px` 的背景模糊、1px 细边框和克制阴影。
- 浅色模式使用冷白玻璃和深色图标；深色模式使用石墨玻璃和浅色图标。强调色继续使用站点现有蓝色。
- 页面为正文预留必要的底部空间，保证最后一段内容不会被按钮遮挡。
- 按压反馈仅做轻微缩放，不使用旋转、弹跳或持续动画。

圆形是悬浮工具的特例，不改变网站现有卡片和普通按钮的圆角规则。

## 展开状态

- 点击按钮后，从底部打开临时目录面板，同时隐藏圆形按钮。
- 面板最大高度约为 `60dvh`，内容不足时按实际高度收缩。
- 面板包含拖动提示、目录标题、章节进度、关闭按钮和可滚动的 H2/H3 列表。
- H3 相对 H2 缩进；当前章节使用浅蓝背景和左侧蓝色标记，不使用高饱和整块按钮。
- 点击章节后更新 URL hash、滚动到对应标题并立即关闭面板。
- 点击遮罩、关闭按钮或按下 `Esc` 均可关闭面板。
- 面板打开时锁定背景滚动，并把焦点移入面板；关闭后焦点返回目录按钮。

## 阅读位置

- 使用 Intersection Observer 跟踪当前 H2/H3，不监听每一帧滚动。
- 当前章节变化时，只更新目录高亮和章节序号，不在正文上显示额外提示。
- 锚点继续沿用现有标题的 `scroll-margin-top`。

## 动效与兼容性

- 面板使用短距离淡入和上移动效，时长保持在约 180–220ms。
- `prefers-reduced-motion: reduce` 下关闭平滑滚动和位移动画。
- 不支持 `backdrop-filter` 时回退为较高不透明度的纯色表面，保证文字对比度。
- 触控目标不小于 44px，焦点样式清晰可见，文本和图标满足 WCAG AA 对比度。

## 验收标准

1. 在 320px、390px 和 430px 宽度下，按钮不遮挡主要正文或系统安全区。
2. 用户从文章任意位置最多一次点击即可打开目录，再一次点击即可跳转章节。
3. 深色和浅色模式使用同一结构，材质与现有站点中性色和蓝色强调保持一致。
4. 键盘、触控、屏幕阅读器和减少动态效果模式均可完成打开、选择和关闭操作。
5. 桌面端目录行为、六语言页面和无目录短文章不发生回归。
