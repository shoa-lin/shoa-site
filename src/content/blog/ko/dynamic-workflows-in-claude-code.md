지난주 Claude Code에 [동적 워크플로](https://code.claude.com/docs/en/workflows)를 출시했습니다. 이제 Claude는 현재 작업에 맞춘 자체 [하네스](https://code.claude.com/docs/en/glossary#agentic-harness)를 즉석에서 작성할 수 있습니다.

> **하네스(Harness)**는 프롬프트 구성, 도구 조율, 컨텍스트 관리, 오류 복구 등 AI 모델을 둘러싼 제어 계층을 뜻합니다. Claude Code는 **Model + Harness**로 이해할 수 있습니다. 이 글에서는 harness를 "하네스"로 표기합니다.

Claude Code의 기본 하네스는 코딩을 위해 설계됐지만, 많은 작업이 본질적으로 코딩 작업과 비슷하기 때문에 다른 여러 작업에도 유용합니다. 다만 [Research](https://support.claude.com/en/articles/11088861-using-research-on-claude), [보안 분석](https://support.claude.com/en/articles/11932705-automated-security-reviews-in-claude-code), [에이전트 팀](https://code.claude.com/docs/en/agent-teams), [Code Review](https://code.claude.com/docs/en/code-review)처럼 최고의 성능을 내려면 Claude Code 위에 맞춤형 하네스가 필요한 작업도 있습니다.

워크플로를 사용하면 Claude가 Claude Code 위에 작업별 하네스를 동적으로 만들 수 있습니다. 이렇게 만든 워크플로는 저장하고 공유하며 재사용할 수도 있습니다.

이 글에서는 워크플로를 사용하며 얻은 초기 경험과 교훈을 소개합니다. 모범 사례는 아직 발전 중입니다. 동적 워크플로는 대체로 더 많은 토큰을 사용하므로 복잡하고 가치가 높은 작업에 가장 적합합니다.

## 프롬프트 예시

기술적인 내용을 살펴보기 전에, 가능한 활용 범위를 보여 주는 몇 가지 프롬프트부터 보겠습니다.

"이 테스트는 약 50번 실행할 때 1번 실패합니다. 이를 재현하는 워크플로를 구성하세요. 경쟁 상태에 대해 서로 다른 가설을 세우고, 증거를 견뎌 내는 가설이 하나 남을 때까지 멈추지 마세요."

"워크플로를 사용해 최근 50개 세션을 살펴보고, 내가 반복해서 지적한 수정 사항을 찾아 자주 등장하는 내용을 `CLAUDE.md` 규칙으로 만드세요."

"워크플로를 사용해 Slack의 #incidents 채널에서 지난 6개월을 조사하고, 반복해서 발생했지만 아무도 티켓을 만들지 않은 근본 원인을 찾으세요."

"내 사업 계획을 대상으로 워크플로를 실행해 서로 다른 에이전트가 투자자, 고객, 경쟁사의 관점에서 철저히 비판하게 하세요."

"여기 이력서 80개가 들어 있는 폴더가 있습니다. 워크플로를 사용해 백엔드 직무에 맞게 순위를 매기고 상위 10명을 다시 확인하세요. AskUserQuestion 도구로 나를 인터뷰해 평가 기준을 정하세요."

"이 CLI 도구의 이름이 필요합니다. 워크플로로 폭넓은 후보를 브레인스토밍한 다음 토너먼트를 실행해 상위 3개를 고르세요."

"워크플로를 사용해 모든 곳에서 User 모델의 이름을 Account로 바꾸세요."

"내 블로그 글 초안을 살펴보고 워크플로를 사용해 모든 기술적 주장을 코드베이스와 대조해 검증하세요. 잘못된 내용은 하나도 게시하고 싶지 않습니다."

## 동적 워크플로의 작동 방식

동적 워크플로는 [서브에이전트](https://code.claude.com/docs/en/sub-agents)를 생성하고 조율하는 몇 가지 특수 함수가 포함된 JavaScript 파일을 실행합니다.

![동적 워크플로가 서브에이전트를 생성하고 조율하는 방식](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a1f1684f559cc83ff4b465b_image1.png)

데이터 처리를 위해 JSON, Math, Array 같은 표준 JavaScript 객체도 제공합니다.

동적 워크플로는 에이전트가 사용할 모델과 서브에이전트를 자체 워크트리에서 실행할지 여부를 선택할 수 있습니다. 이를 통해 Claude는 각 단계에 적합한 지능 수준과 격리 수준을 선택합니다.

사용자 동작이나 터미널 종료로 워크플로가 중단되더라도 세션을 재개하면 멈춘 지점부터 계속할 수 있습니다.

## 동적 워크플로가 필요한 이유

Claude Code의 기본 하네스는 같은 컨텍스트 창에서 작업을 계획하고 실행해야 합니다. 많은 코딩 작업에서는 매우 잘 작동하지만, 장시간 실행되거나 대규모 병렬 처리가 필요하거나 고도로 구조화됐거나 적대적인 작업에서는 한계가 드러날 수 있습니다.

Claude가 하나의 컨텍스트 창에서 복잡한 작업을 오래 수행할수록 다음과 같은 실패 모드에 더 취약해집니다.

- **에이전트의 태만(Agentic laziness)**: Claude가 복잡한 다단계 작업을 끝내기 전에 멈추고 일부만 진행한 상태에서 성공했다고 선언합니다. 예를 들어 보안 리뷰에서 발견 사항 50개 중 35개만 처리할 수 있습니다.
- **자기 선호 편향(Self-preferential bias)**: Claude는 특히 자신의 결과를 평가 기준에 따라 검증하거나 판단하라는 요청을 받았을 때 자기 결과를 선호하는 경향이 있습니다.
- **목표 표류(Goal drift)**: 여러 턴을 거치면서, 특히 컨텍스트 압축 이후에는 원래 목표에 대한 충실도가 점차 낮아집니다. 요약 단계마다 정보가 손실되므로 예외 상황의 요구 사항이나 "X를 하지 말 것" 같은 제약이 사라질 수 있습니다.

워크플로는 각각 자체 컨텍스트 창과 집중된 독립 목표를 가진 별도의 Claude 서브에이전트를 조율해 이러한 실패 모드에 대응합니다.

## 동적 워크플로와 정적 워크플로

Claude Agent SDK나 `claude -p`로 여러 Claude Code 인스턴스를 조율하는 정적 워크플로를 이미 만들어 본 적이 있을 수 있습니다.

정적 워크플로는 모든 예외 상황을 미리 고려해야 하므로 일반적인 형태가 되기 쉽습니다. 반면 [Claude Opus 4.8](https://www.anthropic.com/news/claude-opus-4-8)과 동적 워크플로를 사용하면 Claude가 현재 사용 사례에 맞는 맞춤형 하네스를 작성할 수 있습니다.

## 동적 워크플로에 유용한 패턴

Claude에게 동적 워크플로를 만들라고 간단히 요청하거나, 의도를 명확히 나타내는 트리거 단어 `ultracode`를 사용할 수 있습니다.

자주 쓰이는 패턴을 머릿속에 익혀 두면 워크플로가 유용한 순간을 알아보고 프롬프트로 Claude의 작업 방향을 잡는 데 도움이 됩니다.

Claude는 다음과 같은 패턴을 사용하고 조합할 수 있습니다.

![자주 쓰이는 동적 워크플로 패턴 개요](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a1f16d86247e586b929a407_image10.png)

### 분류 후 실행

분류 에이전트로 작업 유형을 식별한 다음 서로 다른 에이전트나 동작으로 라우팅합니다. 마지막에 분류기를 다시 실행해 결과를 어떻게 처리할지 결정할 수도 있습니다.

### 분산 후 종합

작업을 더 작은 단계로 나누고 각 단계마다 에이전트를 실행한 뒤 결과를 종합합니다. 단계가 많거나, 간섭과 교차 오염을 막는 깨끗한 컨텍스트 창이 각 단계에 도움이 될 때 특히 유용합니다. 종합 단계는 장벽처럼 작동합니다. 분산된 모든 에이전트가 끝날 때까지 기다린 다음 구조화된 출력을 하나의 결과로 합칩니다.

### 적대적 검증

결과를 만드는 에이전트마다 독립 에이전트를 하나 더 생성해 평가 기준이나 명시적 조건에 따라 결과에 이의를 제기하게 합니다.

### 생성 후 필터링

많은 아이디어를 생성하고 평가 기준이나 검증 단계를 통해 거른 뒤 중복을 제거하고, 검증을 통과한 가장 강력한 후보만 반환합니다.

### 토너먼트

작업을 나누는 대신 에이전트끼리 경쟁하게 합니다. 서로 다른 접근 방식으로 같은 작업을 시도하는 N개의 에이전트를 생성한 다음, 하나의 승자가 남을 때까지 심사 에이전트나 모델이 결과를 일대일로 비교하게 합니다.

### 완료될 때까지 반복

작업량을 미리 알 수 없는 경우 고정된 반복 횟수를 정하지 않고, 새로운 발견 사항이 없거나 로그에 오류가 남아 있지 않는 등의 종료 조건을 충족할 때까지 에이전트를 계속 생성합니다.

## 활용 사례

Claude Code에 동적 워크플로를 요청할 시점을 창의적으로 생각해 보세요. 워크플로는 코딩보다 비기술 업무에서 더 유용할 수도 있습니다.

### 마이그레이션과 리팩터링

[Bun](https://bun.com/)은 워크플로를 사용해 Zig에서 Rust로 다시 작성됐습니다. 팀이 이 작업에 어떻게 접근했는지는 [Jarred의 X 스레드](https://x.com/jarredsumner/status/2060050578026189172)에서 확인할 수 있습니다.

핵심은 마이그레이션을 호출 지점, 실패하는 테스트, 모듈 같은 구체적인 단위로 나누는 것입니다. 각 수정 작업마다 워크트리에 서브에이전트를 생성하고, 다른 에이전트가 적대적으로 리뷰하게 한 다음 변경 사항을 병합합니다. 필요하다면 에이전트에게 리소스를 많이 쓰는 명령을 피하도록 지시해 시스템이 더 많은 병렬 작업을 감당할 수 있게 합니다.

### 심층 조사

Claude Code에는 동적 워크플로로 구축된 심층 조사 스킬 `/deep-research`가 포함돼 있습니다. 웹 검색을 분산 실행하고, 출처를 가져오며, 주장을 적대적으로 검증한 다음 인용이 포함된 보고서로 종합합니다.

같은 패턴은 웹 검색 이외의 영역에도 적용됩니다. Claude는 Slack 컨텍스트에서 상태 보고서를 작성하거나 코드베이스를 깊이 탐색해 기능의 작동 방식을 조사할 수 있습니다.

### 심층 검증

![심층 검증 워크플로](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a1f1721824a27cf13da87f4_image2.png)

이미 보고서가 있고 모든 사실 주장을 확인하려면, 한 에이전트가 주장을 식별하고 별도의 서브에이전트가 각 주장을 조사하는 워크플로를 만드세요. 또 다른 검증 에이전트가 인용된 모든 출처가 충분히 신뢰할 만한지 평가할 수도 있습니다.

### 정렬

![정렬 워크플로](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a1f173ce727a972001584cc_image3.png)

버그 심각도순으로 지원 티켓을 정렬하는 것처럼 Claude Code가 평가할 수 있는 정성적 기준에 따라 목록의 순위를 매겨야 한다고 가정해 보겠습니다. 하나의 프롬프트에서 1,000개가 넘는 행을 정렬하려 하면 품질이 떨어지고 유용한 컨텍스트 범위를 초과합니다. 대신 토너먼트, 일대일 비교 에이전트 파이프라인, 또는 병렬 버킷 순위 지정 후 병합하는 방식을 사용하세요. 절대 점수보다 비교 판단이 더 신뢰할 만하며 각 비교에는 독립된 컨텍스트 창이 주어집니다.

### 메모리와 규칙 준수

![메모리와 규칙 준수 워크플로](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a1f17517076bb59050d90bb_image8.png)

규칙이 `CLAUDE.md`에 있는데도 Claude가 반복해서 놓친다면 규칙마다 하나의 검증 에이전트를 두는 워크플로를 만드세요. 회의적인 서브에이전트가 규칙 자체를 검토하게 하면 오탐을 줄일 수 있습니다.

반대 방향도 가능합니다. 최근 세션과 코드 리뷰 댓글에서 반복되는 수정 사항을 찾고, 병렬 에이전트로 군집화하며, 각 후보 규칙이 실제 실수를 막을 수 있었는지 적대적으로 검증한 다음 살아남은 규칙을 `CLAUDE.md`에 다시 정리해 넣을 수 있습니다.

### 근본 원인 조사

디버깅은 여러 독립적인 가설을 세우고 검증할 때 가장 효과적입니다. 하나의 컨텍스트 창에서는 자기 선호 편향이 발생하기 쉽습니다.

워크플로는 서로 겹치지 않는 로그, 파일, 데이터 같은 증거를 별도의 에이전트에 배정해 이를 구조적으로 방지할 수 있습니다. 그런 다음 각 가설을 독립된 검증자와 반박자가 검토하게 합니다.

이 패턴은 코드에만 국한되지 않습니다. 영업 분석, 데이터 엔지니어링 장애, 모든 사후 조사에도 적용됩니다.

### 대규모 분류

![대규모 분류 워크플로](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a1f1778dc00d34cca70819d_image6.png)

모든 팀에는 사람이 전부 처리할 수 없는 지원 대기열, 버그 보고서, 기타 백로그가 있습니다.

분류 워크플로는 각 항목을 분류하고 이미 추적 중인 내용과 중복을 제거한 다음, 수정을 시도하거나 사람에게 에스컬레이션하는 등 적절한 조치를 취합니다.

여기서는 격리(Quarantine) 패턴이 유용합니다. 신뢰할 수 없는 공개 콘텐츠를 읽는 에이전트는 높은 권한이 필요한 작업을 수행하지 못하게 하고, 별도의 에이전트가 해당 정보를 바탕으로 조치하게 합니다.

분류 워크플로를 [`/loop`](https://claude.com/blog/getting-started-with-loops)와 결합하면 Claude가 지속적으로 실행할 수 있습니다.

### 탐색과 취향 판단

디자인이나 이름 짓기처럼 최종 선택이 취향에 달려 있고 그 취향을 평가 기준으로 표현할 수 있다면, 워크플로는 여러 해법을 탐색하는 데 유용합니다.

Claude에게 다양한 선택지를 탐색하게 한 다음 좋은 해법의 평가 기준을 리뷰 에이전트에 제공하세요. 리뷰어가 기준이 충족됐다고 판단하면 작업이 끝납니다. 토너먼트로 평가 기준에 따라 후보의 순위를 매기거나 선택할 수도 있습니다.

### 평가

워크트리에서 독립 에이전트를 생성하고 비교 에이전트가 평가 기준에 따라 출력을 채점하게 해 간단한 평가를 실행할 수 있습니다. 이 방식은 구체적인 기준에 따라 스킬을 평가하고 개선하는 데 사용할 수 있습니다.

### 모델과 지능 수준 라우팅

작업에 맞춘 분류 에이전트를 만들고 모델을 선택하게 합니다. 사전 조사와 도구 호출을 통해 실제 실행에 필요한 지능 수준을 알 수 있는 경우 유용합니다.

예를 들어 "auth 모듈의 작동 방식을 설명하세요"라는 작업에 적합한 모델은 해당 모듈의 규모와 구조에 따라 달라집니다. 분류기는 모듈을 먼저 살펴본 다음 예상 복잡도에 따라 Sonnet이나 Opus로 작업을 라우팅할 수 있습니다.

## 동적 워크플로를 사용하지 말아야 할 때

워크플로는 새로운 기능입니다. 매우 큰 성과를 낼 수 있지만 모든 작업에 필요한 것은 아니며 훨씬 많은 토큰을 사용할 수도 있습니다.

병렬 처리, 전문화, 적대적 검증이 조율 비용만큼 가치가 있을 때 사용하세요. 일반적인 코딩 작업 대부분에는 5명의 리뷰어로 구성된 패널이 필요하지 않습니다. 아키텍처 수준에서 [멀티 에이전트 시스템과 싱글 에이전트 시스템](https://claude.com/blog/building-multi-agent-systems-when-and-how-to-use-them) 중 하나를 선택할 때도 같은 판단이 적용됩니다.

## 동적 워크플로 구축 팁

### 프롬프트 작성

관련 워크플로 패턴을 구체적으로 명시한 자세한 프롬프트가 가장 좋은 결과를 냅니다.

워크플로는 큰 작업에만 국한되지 않습니다. 하나의 가정을 짧게 적대적으로 리뷰하는 "quick workflow"를 요청할 수도 있습니다.

### `/goal` 및 `/loop`와 결합하기

분류, 조사, 검증처럼 반복 가능한 워크플로에는 [`/loop`](https://claude.com/blog/getting-started-with-loops)와 [`/goal`](https://code.claude.com/docs/en/workflows)을 결합하세요. 일정한 간격으로 실행하면서 반드시 충족해야 하는 완료 조건을 설정할 수 있습니다.

### 토큰 사용량 예산

동적 워크플로에 명시적인 토큰 예산을 설정할 수 있습니다. "use 10k tokens" 같은 프롬프트는 작업의 토큰 상한을 10k로 설정합니다.

### 동적 워크플로 저장 및 공유

워크플로 메뉴에서 `s`를 누르면 워크플로가 저장됩니다. `~/.claude/workflows`에 체크인하거나 스킬을 통해 배포할 수 있습니다.

![워크플로 메뉴에서 워크플로 저장하기](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a1f17b1ca20533e666c867c_image4.png)

스킬을 통해 공유하려면 JavaScript 워크플로 파일을 스킬 폴더에 넣고 `SKILL.md`에서 참조하세요. 더 많은 유연성이 필요하다면 Claude에게 워크플로를 반드시 그대로 실행해야 하는 스크립트가 아니라 템플릿으로 취급하라고 지시하세요.

![스킬을 통해 워크플로 공유하기](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a1f17cb835cf4f9fd5da921_image7.png)

## 탐색을 위한 새로운 출발점

워크플로는 Claude Code를 확장하는 유용한 새 방법입니다. Claude가 작업을 도울 새로운 방식을 발견하기 위한 출발점으로 활용하세요. 효과적으로 사용하는 방법은 아직 배울 것이 많습니다.
